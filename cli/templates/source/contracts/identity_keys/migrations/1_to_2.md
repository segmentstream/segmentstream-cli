# Migrate __CONTRACT_TYPE__ From Schema Version __FROM_SCHEMA_VERSION__ To __TO_SCHEMA_VERSION__

The `identity_keys` contract now emits timestamped observation rows. Your source
currently emits daily rows, so update the source package at `__SOURCE_PATH__` to
include the exact observation timestamp that produced each key.

## Required Edits

1. Update `__SOURCE_PATH__/contract.yml`:

```yaml
type: identity_keys
schema_version: 2
model:
  name: identity_keys
  partition: date
```

2. Update `__SOURCE_PATH__/models/identity_keys.sql` so it returns observation
   rows with these required columns:

```sql
select
  date(observed_at) as date,
  observed_at,
  anonymous_id,
  key_name,
  key_value
from ...
```

Use the source event timestamp for `observed_at`. For SDK events this is usually
the event timestamp column, cast to a warehouse timestamp. Do not calculate daily
first or last observations in the source; analytics-core now performs that
compression.

3. Update `__SOURCE_PATH__/models/schema.yml` so the `identity_keys` model
   contract metadata says `schema_version: 2` and documents `observed_at`:

```yaml
models:
  - name: identity_keys
    meta:
      segmentstream:
        contract:
          type: identity_keys
          schema_version: 2
    columns:
      - name: date
      - name: observed_at
      - name: anonymous_id
      - name: key_name
      - name: key_value
```

4. Update `__SOURCE_PATH__/tests/verify_identity_keys_contract.sql` so it fails
   rows where:

```sql
observed_at is not null
date != date(observed_at)
date < date('{{ segmentstream_start_date }}')
date >= date('{{ segmentstream_end_date }}')
```

Keep the `segmentstream_source_verify` tag on all source verification tests.

5. Run:

```bash
segmentstream source verify __SOURCE_NAME__
```

## Fast Path

If the source was mostly scaffold boilerplate, regenerate the latest template and
port only your extraction SQL:

```bash
segmentstream source scaffold <new_source_name> --type identity_keys
```

Then compare the generated `contract.yml`, `models/schema.yml`, and
`tests/verify_identity_keys_contract.sql` against `__SOURCE_PATH__`.
