# Generated SegmentStream Runtime

This directory is generated by `segmentstream init` and `segmentstream run`.

Do not edit files here directly. The CLI may delete and recreate this directory
at any time. Update `../segmentstream.yml` and run `segmentstream run` instead.

Use `segmentstream run` from the project root to regenerate this directory,
restart the local Dagster/dbt runtime, and run materialization.
The first run can take a few minutes while Docker downloads and builds the
local environment; later runs should be faster.

## Files

- `docker-compose.yml` starts the local SegmentStream runtime.
- `Dockerfile` defines the local Python/dbt/Dagster image.
- `dbt_project.yml` and `profiles.yml` are generated from the embedded SegmentStream runtime template.
- `.env` is generated from `segmentstream.yml` and is used by Docker Compose and dbt.
- `dagster/` contains the generated Dagster entrypoint.
- `dbt/`, `target/`, and `logs/` are runtime working directories.

## Debugging

The manual Docker debugging equivalent after generation is:

```sh
cd .segmentstream
docker compose up -d --build --force-recreate
docker compose exec -T segmentstream dbt build --project-dir /workspace/.segmentstream --profiles-dir /workspace/.segmentstream
```
